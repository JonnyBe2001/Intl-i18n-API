Intl (i18n/internationalization) API

Wozu braucht es diese WEB-API

Eine Internationalisierungs-API (i18n-API) ist ein wichtiges Werkzeug für Entwickler, um die internationale Nutzung ihrer Anwendung zu ermöglichen:

  • Mehrsprachige Unterstützung: Texte und Inhalte können in verschiedenen Sprachen verwaltet werden.
  • Anpassung an regionale Unterschiede: Unterschiede in Datums- und Zeitformate, Währungen, Zahlenformate und Schreibweisen können berücksichtigt und angepasst werden.

Beachte: Diese API dient lediglich der Formatierung und beachtet deshalb nicht:

  • Zeitzonen
  • Umrechnung von Währungen
  • Umrechnung von Maßeinheiten
  • Umrechnung von Gewichten
  • ...

Für die Berücksichtigung dieser Eigenschaften müssen andere Web APIs eingebunden werden, wie beispielsweise die Geolocation-API für eine Zeitzonen-Berücksichtigung

    //countryFormat() wird  durch durch den Klick auf einen Button ausgelöst und der zum Button gehörende languageCode mitgegegeben
    function countryFormat(languageCode){
        //Formatierungsangaben des Datums:
        const dateOptions = {weekday: 'long', year: 'numeric', month: 'long', day: 'numeric'};
        //aktuelles Datum:
        const time = new Date();
        //Formatierung des Datums:
        const formattedDate = new Intl.DateTimeFormat(languageCode, dateOptions).format(time);
        //Ausgabe des formatierten Datums:
        document.querySelector("#date").innerHTML=formattedDate;
    }
